# Task 4 - Complex Form Validation & Dynamic DOM Manipulation
Author: Kiruthika

Objective:
- Implement client-side form validation (username, email, optional phone, password strength, confirm password).
- Dynamically update the DOM with registered user cards (including phone).
- Simple client-side routing: Login / Register / Users (no page reload).
- Persist data to localStorage so registered users remain after page reloads.

How to run:
1. Open `index.html` with Live Server in VS Code.
2. Use the Register form to create users (phone optional).
3. View registered users under Users and remove if needed.

Notes:
- Images used are in `assets/images/` (logo.png, register.png, user.png).
- Demo-only: no backend or real authentication; passwords are not stored.
